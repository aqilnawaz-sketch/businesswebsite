In cPanel → Directory Privacy → select the folder containing the file → enable password protection → set a second layer username/password. This adds server-level authentication on top of the app's own password.

Recommended: Use Option B (subdomain) + Option C (directory password) for maximum security. The app will then require two passwords — hosting-level and app-level.

3. WhatsApp Reminders Setup

Service used: CallMeBot — free WhatsApp API. Sends to your personal WhatsApp number 0333-3621111 automatically. Limit: unlimited messages on personal numbers. No subscription, no cost.

One-Time Setup (5 minutes)

1

Save the CallMeBot number

Add this number to your WhatsApp contacts: +34 644 82 47 47. Save as "CallMeBot".

2

Send activation message

Open WhatsApp → open the CallMeBot chat → send this exact message:
I allow callmebot to send me messages

3

Receive your API Key

Within 1 minute, CallMeBot replies with your personal API key. It looks like: 1234567 (a 7-digit number).

4

Enter key in the app

Open MLA Litigation Desk → Reminders page → click "Send WhatsApp Now" → paste your API key when prompted. It is saved automatically — you never need to enter it again.

How Auto-Reminders Work

Every time you open the app (once per calendar day), it automatically checks:

  • Are there hearings today? → Sends WhatsApp immediately
  • Are there hearings tomorrow? → Sends WhatsApp reminder
  • Neither → No WhatsApp sent (no spam)

The WhatsApp message includes: today's & tomorrow's hearings, case name, court, stage, and outstanding fee alerts.

Reset API Key

Click "Reset Key" on the Reminders page. The stored key is deleted. Next time you click Send WhatsApp, you will be prompted to enter it again.

5. Adding & Editing Cases

Currently cases are loaded from the SEED array in the file. To add real cases permanently, either:

Method A — Edit the file directly (for developer)

Open the HTML file in VS Code. Find the const SEED=[ array. Add a new case object following the schema in Section 6. Save and re-upload to hosting.

Method B — Browser localStorage (current behavior)

The app saves all case data to localStorage under the key mla_cases. Data persists on the same browser/device. To export your cases, open browser DevTools (F12) → Console → type:

copy(localStorage.getItem('mla_cases'))

This copies your cases as JSON. You can paste this into the SEED array to make them permanent.

Important: localStorage is device-specific. If you open the app on a different device (phone vs laptop), they will not share case data automatically. To sync: export JSON from one device, import to another by replacing the SEED array.

6. Case Data Schema

Each case in the SEED array must follow this structure:

{
  id:        'c6',                          // unique ID
  no:        '2025/CS/0099',               // case number
  title:     'Client Name vs. Opponent',
  client:    'Client Full Name',
  father:    'Father Name',
  cnic:      '42201-0000000-0',
  mobile:    '0300-0000000',
  address:   'Full address',
  court:     'Court name, City',
  judge:     'Hon. Justice ...',
  cat:       'Criminal',    // Criminal | Civil | Commercial
  stage:     'Bail Hearing', // see stages below
  next:      '2026-06-15',   // YYYY-MM-DD format REQUIRED
  filed:     '2025-01-01',
  prov:      'PPC 302, 34',  // legal provisions
  fir:       'FIR No. 12/2025',
  ps:        'Police Station Name',
  opponent:  'Opponent Name',
  opC:       'Opposing Counsel Name',
  strategy:  'Your litigation strategy notes',
  jnotes:    'Notes about judge behavior',
  otactics:  'Opposing counsel tactics',
  settle:    'Low | Medium | High',
  evwk:      'Evidence weaknesses',
  warn:      'Important warnings',
  priority:  'urgent',        // urgent | high | normal
  risk:      'red',           // green=reliable | yellow=slow | red=problematic
  agreed:    500000,          // numbers only, no ₨ symbol
  received:  250000,
  expense:   15000,
  payments: [
    {date:'2025-01-15', amt:250000, method:'Cash', note:'Retainer'}
  ],
  timeline: [
    {date:'2025-01-15', ev:'Case Filed', type:'filing'}
    // types: filing | hearing | order | evidence | arguments
  ]
}

Use exactly one of these for the stage field — they control the color coding:
